## Описание
Подсистема приема задач из различных источников для платформы обработки задач. Реализация демонстрирует контрактное программирование и duck typing. Архитектура позволяет добавлять новые источники без изменения существующего кода.

## Структура проекта
```text
DuckDuck/
├── src/                    # Исходный код приложения
│   ├── main.py             # Точка входа в приложение и запуск сбора
│   ├── source.py           # Контракт и реализации источников задач
│   ├── task.py             # Модель данных задачи (Task)
├── tests/                  # Модульные тесты
│   ├── test_main.py        # Тесты для точки входа и функции сбора
│   ├── test_source.py      # Тесты для источников задач
│   └── test_task.py        # Тесты для модели задачи
├── log.txt                 # Файл с логами работы приложения
├── requirements.txt        # Конфигурация проекта (зависимости, pytest)
├── tasks.json              # Входной JSON-файл с задачами
└── README.md               # Документация проекта
```

## Функционал
- Единый контракт `TaskSource` для всех источников задач (метод `get_tasks()`).
- Модель задачи `Task`, содержащая уникальный идентификатор `id` и полезную нагрузку `payload`.
- Три источника задач:
  - `FileTaskSource`: чтение задач из JSON-файла.
  - `GeneratorTaskSource`: программная генерация заданного количества задач.
  - `APITaskSource`: API-заглушка с возможностью динамического добавления задач.
- Функция `collect_all_tasks`, которая собирает задачи изо всех источников с предварительной runtime-проверкой на соответствие контракту с помощью `isinstance()`.
- Логирование процесса с использованием библиотеки `loguru` (логи сохраняются в файл `log.txt`).

## Запуск
Для запуска точки входа (сбор задач из всех истоников и вывод в консоль):
```shell
python -m src.main
```

## Тестирование
Для запуска всех тестов выполните команду:
```Shell
python -m pytest tests/
```
